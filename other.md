四、具体功能模块实现思路
1. 润色功能
用户选中一段文字 → 点击"润色" → 后端调用 AI API（附带当前人物设定作为上下文）→ 返回润色结果 → 用户选择替换或修改

进阶：内置"去 AI 味"技能（禁用高频套话、口语化改写）

2. 时间线管理
数据结构：事件 { id, 标题, 描述, 时间(精确日期/模糊如"春天"), 关联人物[], 关联地点[], 章节引用 }

可视化：用 React 组件做时间轴视图，支持筛选和拖拽调整

3. 关系追踪
数据结构：关系 { 人物A, 人物B, 关系类型(师徒/敌对/同盟/血亲等), 描述, 证据章节 }

可视化：用 D3.js 或 Cytoscape.js 做力导向关系图

进阶：AI 自动从文本中提取关系

4. 人物管理
人物卡片：姓名、别名、身份、性格描述、外貌、背景故事、能力、目标、弧光

关联：自动记录人物在哪些章节出现

进阶：跨章节动态状态追踪

5. 情节设计
大纲层级：全书大纲 → 卷 → 章节 → 场景

情节节点：每个节点包含 { 事件, 目标, 情绪基调, 涉及人物 }

可视化：看板或脑图形式

6. 头脑风暴
自由对话式 AI 聊天：用户可以问"帮我构思一个反派"、"主角在这里应该怎么选择"

进阶：AI 主动提问引导思考（类似"苏格拉底式问答"）
