# 任务：完善小说设定

> **额外要求**：请重点完善主角的成长弧线和配角的关系网络

> **重要**：如下是可用字段定义。生成的 operations 必须严格使用下面列出的字段名，不要自创字段。
如果你需要表达一个不在列表中的信息，请在已有字段的 placeholder 方向去发挥，而不是新增字段。

## 角色 (Character) 可用字段

| 字段名 | 标签 | 类型 | 说明 |
|--------|------|------|------|
| `name` | 角色名称 | text | 角色名称 |
| `gender` | 性别 | single | 可选值: 男 / 女 |
| `role` | 角色类型 | single | 可选值: 主角 / 配角 / 反派 |
| `age` | 年龄 | text | 如：25岁 |
| `identity` | 身份 | text | 身份/职业/组织归属 |
| `item` | 标志性物件 | text | 一件有故事的随身物品 |
| `coreConflict` | 核心矛盾 | text | 一句话概括 |
| `surfaceGoal` | 表层目标 | text | 角色表面追求的目标... |
| `deepNeed` | 深层需求 | text | 角色内心真正渴望的... |
| `emotionExpression` | 情感表达方式 | single | 可选值: 成熟克制 / 笨拙 / 热烈 / 压抑 / 回避 |
| `abilities` | 能力 | tags |  |
| `growthArcs` | 成长弧光 | list | 子字段: [缺陷, 表现, 成长方向] |
| `notes` | 写作注意事项 | list | 子字段: [标签, 内容] |
| `experience` | 人物经历 | list | 子字段: [经历描述] |
| `narrativeFunction` | 叙事功能原型 | multi | 可选值: 英雄 / 导师 / 信使 / 变形者 / 阴影 / 伙伴 / 骗徒 / 边界守卫；最多选 3 项 |
| `innerMotivation` | 内在动机原型 | multi | 可选值: 天真者 / 孤儿 / 战士 / 照顾者 / 探索者 / 爱人 / 叛逆者 / 创造者 / 愚者 / 统治者 / 魔术师 / 贤者；最多选 3 项 |

### 字段类型示例

```json
{
  "name": "示例文本", // text：普通字符串
  "gender": "选项值", // single：从上方可选值中选择一个
  "narrativeFunction": ["选项1", "选项2"], // multi：从上方可选值中选择多项，JSON数组
  "abilities": ["标签1", "标签2"], // tags：自由标签，JSON数组
  "growthArcs": ["缺陷;表现;成长方向", "缺陷;表现;成长方向"], // list：数组，每项内部用分号分隔子字段
}
```


## 组织/势力 (Organization) 可用字段

| 字段名 | 标签 | 类型 | 说明 |
|--------|------|------|------|
| `name` | 组织名称 | text | 组织名称 |
| `description` | 描述 | longtext | 组织的简要描述 |
| `type` | 组织类型 | multi | 可选值: 情报组织 / 研究机构 / 犯罪集团 / 宗教组织 / 政府机构 / 商业集团 / 佣兵团 / 学院门派 / 秘密结社 / 反抗军；最多选 2 项 |
| `status` | 状态 | single | 可选值: 活跃 / 蛰伏 / 已解散 |
| `foundingBackground` | 成立背景 | longtext | 组织成立的背景与契机 |
| `founder` | 创始人 | text | 创始人姓名 |
| `currentLeader` | 当前负责人 | text | 当前负责人姓名 |
| `operationLogic` | 运行逻辑 | longtext | 组织的运作方式和逻辑 |
| `structure` | 结构 | list | 子字段: [等级, 作用/职责] |
| `headquarters` | 总部/据点 | tags |  |
| `members` | 成员 | list | 子字段: [代号, 等级, 备注] |
| `opponents` | 对手 | tags |  |
| `mortalEnemies` | 死敌 | tags |  |
| `allies` | 盟友 | tags |  |

### 字段类型示例

```json
{
  "name": "示例文本", // text：普通字符串
  "description": "较长文本...", // longtext：长文本字符串
  "status": "选项值", // single：从上方可选值中选择一个
  "type": ["选项1", "选项2"], // multi：从上方可选值中选择多项，JSON数组
  "headquarters": ["标签1", "标签2"], // tags：自由标签，JSON数组
  "structure": ["等级;作用/职责", "等级;作用/职责"], // list：数组，每项内部用分号分隔子字段
}
```

## 输出要求

你必须以**严格 JSON** 格式输出，不要包裹在 markdown 代码块中。输出格式为一个 JSON 对象：

```json
{
  "analysis": "整体分析简述，说明本次完善的核心思路",
  "operations": Operation[]
}
```

### Operation 类型定义

每个 operation 包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| changeType | "add" \| "update" \| "delete" | 操作类型 |
| reason | string | 执行原因，限制200字以内 |
| summary | string | 一句话概括操作内容 |
| api | string | API 路径，见下方 API 路径对照表 |
| method | "POST" \| "PUT" \| "DELETE" | HTTP 方法 |
| params | object | 根据 changeType 和 api 参照上方字段模板填写 |

### Operation 示例（add / update / delete）

**新增 (add)**：params 需包含 novelId + name + 要设置的字段
```json
{
  "changeType": "add",
  "reason": "新增角色的原因",
  "summary": "一句话概括",
  "api": "/api/characters",
  "method": "POST",
  "params": {
    "novelId": "mrp2zihp9397xf",
    "name": "角色名",
    ...其他字段（参照上方字段表和字段类型示例）
  }
}
```

**更新 (update)**：params 需包含 novelId + id + 要修改的字段（一次只改一个字段）
```json
{
  "changeType": "update",
  "reason": "修改字段的原因",
  "summary": "一句话概括",
  "api": "/api/characters",
  "method": "PUT",
  "params": {
    "novelId": "mrp2zihp9397xf",
    "id": "实体的id（从上文实体表格的 id 列获取）",
    "字段名": "新值"
  }
}
```

**删除 (delete)**：params 需包含 novelId + id
```json
{
  "changeType": "delete",
  "reason": "删除实体的原因",
  "summary": "一句话概括",
  "api": "/api/characters",
  "method": "DELETE",
  "params": {
    "novelId": "mrp2zihp9397xf",
    "id": "要删除的实体id（从上文实体表格的 id 列获取）"
  }
}
```

- **novelId**：固定为 `"mrp2zihp9397xf"`，所有操作都必须使用这个值
- **实体 id**：每个实体的唯一标识，从上文实体表格的 id 列获取

### API 路径对照

| 实体 | API 路径 |
|------|----------|
| 角色 | /api/characters |
| 组织/势力 | /api/organizations |
| 地点 | /api/locations |
| 关系 | /api/relations/link |

### 重要规则

1. **字段名约束**：只使用"可用字段"列表中定义的字段名，不要自创字段
2. **字段值约束**：single 类型须用可选值参考中的选项；multi/tags 用 JSON 数组；list 用字符串数组（分号分隔）
3. **update 粒度**：一个操作只改一个字段
4. **必须输出纯 JSON**：不要用 markdown 代码块包裹，不要加额外说明文字

**数量要求**：总共生成 5~8 条操作建议，优先修正逻辑矛盾、补充主角关键设定，至少要包含一个地点类型

**reason理由**：限制在200字以内
