## 设计理念

琥珀色彩系统建立在一个**极深暖灰**的底色之上，以**琥珀金**作为唯一强调色。整体氛围偏向古典、叙事感，避免了常见的冷色调科技感。

三个核心原则：

1. **克制** —— 全局仅一个强调色系，通过明暗变化承担所有强调职能
2. **层次** —— 背景从 `#0f0e0c` 到 `#e8e0d0` 共 12 级，确保每个层级都有清晰的视觉区分
3. **温度** —— 所有灰色都带有微暖色调（HSL 色相约 30°-40°），杜绝冷灰

## 文字色 · Text Hierarchy

文字色从背景色阶中提取 4 级，加上语义色的浅色变体，构成完整的文字层级。

| Token | HEX | 对比度 (on bg-950) | WCAG 等级 | 使用场景 |
|-------|-----|-------------------|-----------|----------|
| `--fg-primary` | `#e8e0d0` | **12.8:1** | AAA | 标题、正文、重要信息 |
| `--fg-secondary` | `#9a9080` | **5.6:1** | AA | 描述文字、辅助说明 |
| `--fg-tertiary` | `#6a6258` | **3.2:1** | — | 标签名、提示文字、占位符 |
| `--fg-disabled` | `#4a4438` | **1.8:1** | — | 禁用态文字（仅装饰性） |

## 语义色 · Semantic Colors

四组语义色，每组包含浅色（文字/图标）、标准色（背景/边框）、深色（悬停态）三级。

|-------|-----|------|

```
✅ 正确用法

❌ 错误用法
   用 --danger 作为页面主色调
   用 --success-light 作为大段正文色
   同时使用 3 种以上语义色
```

## 渐变 · Gradients

### 主色渐变

```css
/* 按钮主渐变 —— 从主色到深色，由左上到右下 */
background: linear-gradient(135deg, var(--accent-500), var(--accent-600));

/* 文字渐变 —— 从浅到深 */
background: linear-gradient(135deg, var(--fg-primary) 30%, var(--accent-400) 100%);
-webkit-background-clip: text;
-webkit-text-fill-color: transparent;

/* 头像光环 —— 三段式渐变 */
background: linear-gradient(135deg, rgba(200,149,108,0.4), rgba(160,107,62,0.2), rgba(87,83,78,0.3));
```

### 分隔线渐变

```css
/* 水平渐隐线 */
height: 1px;
background: linear-gradient(90deg, transparent, var(--border-default), transparent);

/* 顶部强调线 */
height: 1px;
background: linear-gradient(90deg, transparent, var(--accent-500), transparent);

/* 左侧强调条 —— 竖向渐变 */
width: 4px;
background: linear-gradient(180deg, var(--accent-500), var(--accent-700));
```

### 装饰光晕

```css
/* 右上角光斑 */
background: radial-gradient(ellipse, rgba(200,149,108,0.06) 0%, transparent 70%);

/* 左下角光斑 */
background: radial-gradient(ellipse, rgba(160,107,62,0.05) 0%, transparent 70%);
```

### 渐变使用规则

```
✅ 允许
   · 主色 → 主色深色（按钮、强调元素）
   · 透明 → 实色（分隔线、光晕）
   · 文字渐变仅用于页面标题

❌ 禁止
   · 多色渐变（如：琥珀 → 蓝色 → 绿色）
   · 高饱和度渐变
   · 在小元素上使用复杂渐变
   · 背景渐变角度超过 180° 范围
```

## 对比度与可访问性

### WCAG 合规矩阵

| 前景色 | 背景色 | 对比度 | WCAG | 等级 |
|--------|--------|--------|------|------|
| `--fg-primary` (#e8e0d0) | `--bg-950` (#0f0e0c) | **12.8:1** | AAA | ✅ 正文可用 |
| `--fg-primary` (#e8e0d0) | `--bg-700` (#211f1a) | **9.2:1** | AAA | ✅ 正文可用 |
| `--fg-secondary` (#9a9080) | `--bg-950` (#0f0e0c) | **5.6:1** | AA | ✅ 正文可用 |
| `--fg-secondary` (#9a9080) | `--bg-700` (#211f1a) | **4.0:1** | AA | ✅ 大字可用 |
| `--fg-tertiary` (#6a6258) | `--bg-950` (#0f0e0c) | **3.2:1** | — | ⚠️ 仅装饰文字 |
| `--accent-400` (#daa882) | `--bg-950` (#0f0e0c) | **7.8:1** | AAA | ✅ 正文可用 |
| `--accent-500` (#c8956c) | `--bg-950` (#0f0e0c) | **5.9:1** | AA | ✅ 正文可用 |
| `--accent-500` (#c8956c) | `--bg-700` (#211f1a) | **4.2:1** | AA | ✅ 大字可用 |
| `--success-light` (#a8d5ab) | `--bg-950` (#0f0e0c) | **10.1:1** | AAA | ✅ 正文可用 |
| `--danger-light` (#e89898) | `--bg-950` (#0f0e0c) | **6.8:1** | AA | ✅ 正文可用 |
| `--fg-disabled` (#4a4438) | `--bg-950` (#0f0e0c) | **1.8:1** | — | ❌ 不可用于任何文字 |

### 关键规则

```
1. 所有正文文字必须达到 WCAG AA（≥ 4.5:1）
2. 大号文字（≥ 18px / 14px 加粗）可放宽至 ≥ 3:1
3. --fg-tertiary 仅用于标签名、图标旁注等非关键信息
4. --fg-disabled 绝对不可用于任何需要阅读的文字
5. 语义浅色（-light 后缀）在深底色上均达标，可放心使用
6. 主色文字 --accent-400 优先于 --accent-500，对比度更高
```

## 使用规则

### 色彩角色分配

```
┌──────────────────────────────────────────────────┐
│                    角色分类                        │
├──────────┬───────────────────────────────────────┤
│ 唯一强调  │  --accent-500 系列                     │
│          │  所有交互反馈、选中、高亮都用这一组       │
├──────────┼───────────────────────────────────────┤
│ 结构底色  │  --bg-950 ~ --bg-700                   │
│          │  通过明度差异区分层级，不引入新色相        │
├──────────┼───────────────────────────────────────┤
│ 语义状态  │  success / warning / danger / info      │
│          │  仅用于状态反馈，不作为装饰色             │
├──────────┼───────────────────────────────────────┤
│ 文字层级  │  --fg-primary / secondary / tertiary   │
│          │  严格按层级使用，不跨级                  │
└──────────┴───────────────────────────────────────┘
```

### 禁止事项

```
❌ 绝对禁止
   1. 引入蓝色、紫色、绿色等新色相作为主色
   2. 使用纯白 #ffffff 或纯黑 #000000
   3. 使用冷灰色（HSL 色相 < 20° 或 > 50° 的灰色）
   4. 同时高亮超过 3 种语义色
   5. 在 --bg-700 以上的底色上使用 --fg-disabled

⚠️ 谨慎使用
   1. --accent-100 ~ --accent-200 仅在极浅底色上使用
   2. --bg-100 作为文字色仅用于装饰性大标题
   3. 语义色深色变体（-deep）仅用于悬停态，不作为静态色
```

### 配色决策树

```
需要为一个元素选色？
│
├─ 是文字？
│  ├─ 标题 / 正文        → --fg-primary
│  ├─ 描述 / 辅助说明     → --fg-secondary
│  ├─ 标签名 / 提示       → --fg-tertiary
│  └─ 禁用态             → --fg-disabled
│
├─ 是背景？
│  ├─ 页面底色           → --bg-950
│  ├─ 导航 / 面板         → --bg-800
│  ├─ 卡片               → --bg-700
│  ├─ 输入框             → --bg-850
│  └─ 需要强调的背景       → rgba(200,149,108, 0.08~0.15)
│
├─ 是边框？
│  ├─ 静态默认           → --border-subtle / --border-default
│  ├─ 悬停               → --border-strong
│  ├─ 聚焦               → --border-focus + shadow
│  └─ 选中 / 强调         → --border-accent
│
├─ 是交互状态？
│  ├─ 主按钮             → gradient(accent-500 → accent-600)
│  ├─ 次要按钮           → accent-10 背景 + border-accent 边框
│  ├─ 幽灵按钮           → 透明 + border-default
│  └─ 危险操作           → danger-10 背景 + danger 边框
│
└─ 是状态反馈？
   ├─ 成功               → success 系列
   ├─ 警告               → warning 系列
   ├─ 错误               → danger 系列
   └─ 信息               → info 系列
```



### 悬停态必须可见但不过分

```css
/* 推荐的悬停处理方式 —— 同时调整边框和背景 */

  transition: all 0.35s var(--ease-spring);
}

.card:hover {
  background: var(--bg-600);        /* 亮一级 */

  transform: translateY(-2px);       /* 微位移 */
  box-shadow: var(--shadow-lg);      /* 加阴影 */
}
