# 产品上下文

## 为什么做这个项目？

现有写作工具缺乏专门为小说创作设计的结构化支持。作者需要同时管理人物设定、章节进度、时间线一致性等多维度信息，通用工具难以满足。

## 解决了什么问题？

1. **信息碎片化**：人物、情节、设定分散在不同文档中
2. **一致性维护难**：长篇创作中容易忘记设定、产生前后矛盾
3. **写作效率低**：缺乏专注的写作环境和便捷的 AI 辅助
4. **AI 辅助无定制**：通用 AI 工具不懂小说创作，规则不可配置

## 核心使用场景

1. **日常写作**：选择章节 → 编辑器写作（3s 防抖自动保存，字数统计，Tab 缩进）
2. **人物管理**：点击人物 → 右侧抽屉展开 → 编辑特征数据（自动保存）
3. **章节组织**：左侧列表排序/筛选 → 点击切换编辑
4. **AI 润色**：选中文本 → 浮动菜单 → 选择规则 → AI 结果 → 确认替换/取消
5. **作品概览**：预览模式 → 点击编辑 → 修改信息 → 保存

## 设计原则

1. **本地优先**：所有数据存本地 SQLite，AI 调用外无网络依赖
2. **简洁高效**：最小化 UI 干扰，深色琥珀主题减少视觉疲劳
3. **渐进增强**：基础功能立即可用，高级功能按需启用
4. **数据一致性**：Prisma 作为唯一数据源，级联删除保障引用完整性

## 功能优先级

| 功能                                    | 优先级 |
| --------------------------------------- | ------ |
| 章节编辑器、AI 润色、人物管理、作品概览 | P0     |
| 故事大纲、AI 续写、AI 扩写              | P1     |
| 情感追踪、头脑风暴                      | P2     |
| 关系图谱、时间线、导出功能              | P3     |
