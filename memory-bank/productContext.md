# 产品上下文

## 为什么做这个项目？
现有写作工具（如 Word、Notion）缺乏专门为小说创作设计的结构化支持。作者需要同时管理人物设定、章节进度、时间线一致性等多维度信息，通用工具难以满足这些需求。

## 解决了什么问题？
1. **信息碎片化**：人物、情节、设定分散在不同文档中
2. **一致性维护难**：长篇创作中容易忘记设定、产生前后矛盾
3. **写作效率低**：缺乏专注的写作环境和便捷的 AI 辅助
4. **复盘困难**：缺乏版本对比和创作历史追踪

## 核心使用场景
1. **日常写作**：专注的富文本编辑器，自动保存防丢失
2. **人物管理**：结构化的人物卡片，支持标签分类和快速检索
3. **章节组织**：拖拽排序，关联人物标记
4. **AI 润色**：选中文本 → 上方弹菜单 → 点击润色 → 右侧面板展开选择规则 → AI 改写 → 结果浮层可编辑 → 确认替换/取消
5. **作品概览**：一眼看全所有作品、章节进度、人物数量

## 设计原则
1. **本地优先**：所有数据存本地 SQLite，AI 调用外无网络依赖
2. **简洁高效**：最小化 UI 干扰，让作者专注于写作
3. **渐进增强**：基础功能立即可用，高级功能按需启用
4. **模块化**：功能模块解耦，便于独立开发和测试
